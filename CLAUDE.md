# Baseplate Mobile — Claude Code Context

AI hackathon project. React Native + Expo, 5-person team, fast iteration.

## Stack

- **Expo SDK 51** + **Expo Router v3** (file-based routing)
- **NativeWind v4** (Tailwind CSS classes in React Native)
- **Zustand** for global state
- **Anthropic API** via fetch (model: `claude-sonnet-4-6`)
- **TypeScript strict mode**

## File & Folder Breakdown

### Root Config

| File | What it does |
|------|-------------|
| `package.json` | Dependencies + scripts. `main: "expo-router/entry"` is what wires Expo Router as the app entry point instead of `App.tsx`. Scripts: `start`, `lint`, `format:check`, `type-check`. |
| `app.json` | Expo project config. Sets the app `name`, `slug`, `scheme` (deep-link prefix), enables `typedRoutes` (typed `href` in Expo Router), and registers the `expo-router` plugin. |
| `tsconfig.json` | Extends `expo/tsconfig.base`, adds `strict: true` and a `@/*` path alias so imports read `@/components/ui/Button` instead of `../../components/ui/Button`. |
| `babel.config.js` | Two presets: `babel-preset-expo` with `jsxImportSource: "nativewind"` (injects NativeWind's JSX transform) and `nativewind/babel` (processes `className` props at build time). Both are required for NativeWind v4. |
| `metro.config.js` | Wraps Expo's default Metro config with `withNativeWind`, pointing it at `global.css` as the CSS entry. This is what makes `@tailwind` directives resolve at bundle time. |
| `tailwind.config.js` | Content paths (`app/**`, `components/**`), `nativewind/preset`, and full theme extension: every color token references a CSS variable (e.g. `"var(--color-red-500)"`), font families mapped to exact `@expo-google-fonts` export names, border radius tokens. |
| `global.css` | The design system source of truth. Defines all CSS custom properties in `:root`: full rainbow color scale (red→purple, 100–900), gray 50–950, semantic aliases (`--color-primary`, `--color-background`, etc.), font name variables, spacing scale, radius scale. Includes a `@media (prefers-color-scheme: dark)` block that overrides semantic tokens for dark mode automatically. |
| `nativewind-env.d.ts` | Single-line `/// <reference types="nativewind/types" />`. Without this, TypeScript doesn't know that `className` is a valid prop on React Native core components. |
| `.eslintrc.js` | ESLint config: `eslint-config-expo` base + `@typescript-eslint/recommended`. Rules: hooks exhaustive-deps as warn, no unused vars (ignores `_` prefix), no explicit `any` as warn. Ignores `node_modules/`, `.expo/`, build files. |
| `.prettierrc` | Single quotes, 2-space indent, 100-char line width, trailing commas in ES5 positions. All 5 teammates get identical formatting automatically. |
| `.env.example` | Template committed to git. Contains `EXPO_PUBLIC_ANTHROPIC_API_KEY=sk-ant-api03-...`. Copy to `.env` (gitignored) and fill in. The `EXPO_PUBLIC_` prefix is required by Expo to expose the variable to the RN bundle — without it the value is `undefined`. |
| `.gitignore` | Ignores `node_modules/`, `.expo/`, `dist/`, `.env*`, key/cert files, and the native `android/` + `ios/` folders (generated by `expo prebuild`, not committed). |

### `app/` — Screens (Expo Router)

Every `.tsx` file in `app/` is automatically a route. No route registration needed.

| File | Route | What it does |
|------|-------|-------------|
| `app/_layout.tsx` | (root) | Root layout. Imports `global.css` (required for NativeWind CSS variables to load). Calls `useFonts()` to register Plus Jakarta Sans (4 weights), Syne (2 weights), and JetBrains Mono. Shows a blank loading view until fonts are ready — prevents FOUT on first render. Wraps the app in `SafeAreaProvider` and renders a `<Stack>` navigator. |
| `app/index.tsx` | `/` | Immediately `<Redirect>`s to `/(tabs)/home`. Acts as the canonical entry point so there's no blank screen on launch. |
| `app/(tabs)/_layout.tsx` | `/` (tab shell) | Defines the tab bar: active tint `#4f46e5` (indigo-600), inactive `#9ca3af` (gray-400), white background. `(tabs)` is a route group — the parentheses mean it doesn't appear in the URL path. Add new tabs here with `<Tabs.Screen name="..." />`. |
| `app/(tabs)/home.tsx` | `/home` | Home screen. Displays app name in Syne display font, subtitle, and a Card with a button to navigate to the chat screen. Good starting point for adding feature cards. |
| `app/(tabs)/chat.tsx` | `/chat` | Full AI chat UI. Reads `messages`, `isLoading`, `error`, and `sendMessage` from `useAIStore`. Renders a scrollable message list (user messages right-aligned indigo, assistant messages left-aligned surface), a `Skeleton` loader while waiting, an error banner on failure, and a multi-line `TextInput` + Send button at the bottom. Uses `KeyboardAvoidingView` so the input stays above the keyboard on iOS and Android. |

### `components/ui/` — Primitive UI Components

Owned source files, shadcn/reusables-style. Modify them freely — no third-party API to fight.

| File | Props / Variants | What it does |
|------|-----------------|-------------|
| `Button.tsx` | `variant`: default · outline · ghost · destructive<br>`size`: sm · md · lg<br>`loading`: boolean | Renders a `TouchableOpacity` with NativeWind classes derived from variant + size maps. Disabled + loading both apply `opacity-50`. Loading state shows an `ActivityIndicator` inline. Accepts all `TouchableOpacityProps` so `onPress`, `disabled`, etc. pass through. |
| `Card.tsx` | — | `Card` is a `View` with surface background, rounded-xl corners, and border. Sub-components `CardHeader`, `CardTitle`, `CardContent`, `CardFooter` are optional layout slots — use all or just `Card` with raw children. |
| `Input.tsx` | `label`, `error`, `hint` | Controlled `TextInput` with optional label above and error/hint text below. Border changes to `border-destructive` on error and `border-primary` on focus (tracked with local `useState`). Passes all `TextInputProps` through. |
| `Badge.tsx` | `variant`: default · secondary · success · warning · destructive | Small status chip. Each variant maps to a light background + dark text color in the same hue family (e.g. `bg-green-100` + `text-green-700`). `self-start` so it doesn't stretch full width. |
| `Avatar.tsx` | `src`, `name`, `size`: sm · md · lg · xl | Shows an `Image` if `src` is provided. Falls back to a circle with initials derived from `name` (first letter of each word, max 2). Default indigo background for the initials fallback. |
| `Skeleton.tsx` | `className` (any dimensions) | `Animated.View` that pulses opacity 0.4→1→0.4 on loop using RN's native animation driver. `SkeletonText` renders N lines of skeletons, last line at 60% width (matches natural text endings). Use this anywhere you're waiting on AI responses. |
| `Sheet.tsx` | `visible`, `onClose`, `title`, `snapHeight` | Bottom sheet modal. Uses `Animated.spring` on `translateY` for the slide-up/down — no third-party dependency needed. A semi-transparent backdrop is a `TouchableOpacity` that calls `onClose` on tap. `snapHeight` defaults to 50% of screen height; pass a number (px) to customize. |

### `services/`

| File | What it does |
|------|-------------|
| `services/ai.ts` | Single exported function `sendChatMessage(messages, systemPrompt?)`. Sends a `POST` to `https://api.anthropic.com/v1/messages` with the `claude-sonnet-4-6` model, `max_tokens: 1024`, your API key from `ENV`, and the message array. Throws a descriptive error with HTTP status if the response isn't 2xx. Returns the assistant's text string. **Edit this file** to add streaming, change the model, adjust `max_tokens`, or add tools. |

---

### `store/`

| File | What it does |
|------|-------------|
| `store/aiStore.ts` | Zustand store that owns all AI conversation state. **State**: `messages: Message[]`, `isLoading: boolean`, `error: string \| null`, `systemPrompt: string`. **Actions**: `sendMessage(content)` — appends user message, calls `services/ai.ts`, appends assistant message, handles errors; `clearMessages()` — resets the conversation; `setSystemPrompt(prompt)` — changes the system prompt without clearing history. Import with `useAIStore()` in any component. |

---

### `types/`

| File | What it does |
|------|-------------|
| `types/index.ts` | `Message` — the app-level message shape (`id`, `role`, `content`, `createdAt: Date`). `AnthropicMessage` — the wire format sent to the API (`role`, `content` only — no id or date). The store converts between them so the API layer stays clean. |

---

### `constants/`

| File | What it does |
|------|-------------|
| `constants/env.ts` | Reads `process.env.EXPO_PUBLIC_ANTHROPIC_API_KEY` and exports it as `ENV.ANTHROPIC_API_KEY`. Throws at module load time if the key is missing — you'll see a clear error message instead of a silent `undefined` key causing a confusing 401. |

### `.github/workflows/` — CI

All three workflows trigger on `push` to `main` and all pull requests. They run in parallel (independent jobs).

| File | Command | Fails if |
|------|---------|----------|
| `lint.yml` | `npm run lint` | Any ESLint rule violation in `app/`, `components/`, `services/`, `store/`, `hooks/`, `constants/`, `types/` |
| `prettier.yml` | `npm run format:check` | Any file in the repo has formatting that doesn't match `.prettierrc` — catches tabs vs spaces, missing semicolons, etc. |
| `typecheck.yml` | `npm run type-check` (`tsc --noEmit`) | Any TypeScript type error, including strict-mode violations |

## Adding Things

| Task | Where |
|------|-------|
| New screen | Create `app/(tabs)/yourscreen.tsx` |
| New tab | Add `<Tabs.Screen>` in `app/(tabs)/_layout.tsx` |
| New AI feature | Edit `services/ai.ts` + `store/aiStore.ts` |
| New global state | Add to `store/aiStore.ts` or create `store/yourStore.ts` |
| New UI component | Add to `components/ui/` following existing pattern |
| Change AI system prompt | `useAIStore.getState().setSystemPrompt('...')` |
| Change AI model | Edit `MODEL` constant in `services/ai.ts` |

## Design Tokens (global.css)

All live in `:root`. Full rainbow: `--color-{red|orange|yellow|green|teal|blue|indigo|violet|purple}-{100–900}`.
Gray scale: `--color-gray-{50|100|200|300|400|500|600|700|800|900|950}`.
Semantic (prefer these in components): `--color-primary`, `--color-primary-foreground`, `--color-background`, `--color-surface`, `--color-surface-elevated`, `--color-border`, `--color-text`, `--color-text-secondary`, `--color-muted`, `--color-destructive`, `--color-success`, `--color-warning`.
Dark mode overrides the semantic tokens automatically via `@media (prefers-color-scheme: dark)`.

## Key Decisions

- **Expo Router** over React Navigation — adding a screen = creating a file, zero config
- **Zustand** over Context — zero boilerplate, async actions work out of the box
- **fetch over Anthropic SDK** — no Node.js polyfill issues in the React Native Metro bundler
- **NativeWind v4** — shared Tailwind vocabulary across all 5 teammates
- **Plus Jakarta Sans + Syne** over Inter — intentional, distinctive; Inter is overused in AI-generated UIs
